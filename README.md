# CutCam

한 번의 촬영으로 설정한 초 단위마다 자동으로 컷이 나뉘고, 촬영이 끝나면 현장음을 유지하거나 BGM으로 교체해서 하나의 영상으로 만들어주는 개인용 웹앱입니다.

## 파일 구성
- `index.html` — 앱 전체 (촬영 설정 → 카메라 → 완성)
- `manifest.json`, `sw.js` — 홈 화면 아이콘 설치용 PWA 설정
- `icon-192.png`, `icon-512.png` — 앱 아이콘 (임시 아이콘, 원하면 교체 가능)

## 배포 방법 (기존에 쓰시던 방식 그대로)

### GitHub Pages
1. 새 저장소 만들기 (예: `cutcam`)
2. 이 폴더 안의 파일 전부 업로드
3. Settings → Pages → Branch를 `main`으로 설정
4. `https://<username>.github.io/cutcam/` 로 접속 확인

### Netlify
1. Netlify에서 새 사이트 생성, 이 폴더 그대로 드래그 앤 드롭 배포
2. 발급된 주소로 접속 확인

**중요**: 카메라/마이크 권한은 HTTPS에서만 동작해요. GitHub Pages, Netlify 둘 다 기본적으로 HTTPS라 문제없습니다.

## 아이폰 홈 화면에 추가하기
1. Safari로 배포된 주소 접속 (꼭 Safari여야 해요, 크롬은 안됨)
2. 하단 공유 버튼 → "홈 화면에 추가"
3. 홈 화면 아이콘 눌러서 실행하면 브라우저 주소창 없이 앱처럼 열림

## 사용 방법
1. 컷 길이(초), 필터 선택 → 촬영 시작
2. 녹화 버튼을 누르면 설정한 시간마다 자동으로 컷이 나뉘며 계속 촬영됨
3. 앱을 끄지 않는 한 나중에 다시 촬영해도 같은 프로젝트에 이어붙음 (앱 재실행 시 "이어 찍기" 옵션으로 복귀)
4. 종료 버튼 → 현장음 유지 또는 BGM 교체 선택 → "영상 만들기"
5. 완성된 영상은 미리보기 후 저장/공유 (iOS 공유 시트로 사진 앱에 저장 가능)

## 알아두면 좋은 점
- 필터는 iOS 사진 앱 필터(Vivid, Dramatic, Mono, Silvertone, Noir 등)를 CSS로 최대한 비슷하게 재현한 것입니다. iOS 시스템 필터 자체를 웹에서 그대로 가져다 쓸 수는 없어서, 색감을 근사치로 구현했어요.
- 컷 연결 처리는 브라우저 안에서 ffmpeg.wasm으로 진행돼요. 컷 개수가 많거나 영상이 길면 병합에 시간이 좀 걸릴 수 있습니다 (서버로 업로드하지 않고 폰/브라우저 안에서 전부 처리되는 방식이라 느릴 수 있는 대신 개인정보가 밖으로 안 나가요).
- 첫 실행 시 카메라/마이크 권한을 반드시 허용해야 정상 동작합니다.
